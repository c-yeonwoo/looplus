## 3. 핵심 = 재테크 엔진 (`/engine`)

가장 공들인 화면. 팔레트(좌, 접기 가능) → 캔버스(중앙, SVG 흐름 그래프) → 인스펙터(우측, 노드 클릭시만 오버레이) → 결과 패널(캔버스 바로 아래, 동일 너비) 순서.

**흐름 그래프**: 멀티 수입(근로·자본·부수입·프리랜서) → 비율대로 각 버킷에 분산 배분(링크가 서로 겹치지 않게 출구/입구 지점을 분산 라우팅) → 자산 풀. 골드 파티클이 실제로 흘러가는 애니메이션(SMIL, `prefers-reduced-motion` 대응). eCharts 같은 외부 차트 라이브러리 없이 순수 SVG로 구현 — 파티클·클릭·드롭을 완전히 제어하기 위한 선택.

**하이브리드 복리 재투입** (계산 로직의 심장, `src/lib/engine/projection.ts`):
- 미실현 수익(주가·시세) = 해당 버킷/풀 내부 복리
- 실현 자본소득(배당·임대·이자) = 다음 해 upstream 수입으로 재유입 → 재배분
- 연금·IRP 등 `locked` 버킷은 실현분도 잠긴 채 내부 복리(별도 집계, passive에 안 잡힘)
- → **자본소득이 근로소득을 넘어서는 교차 연도(crossover)** 가 자동으로 산출됨

**8단계 신호 게이트 판정** (`src/lib/engine/stage.ts`): 절대 금액이 아니라 저축률·비상금·투자시작·passive발생 등 신호 누적으로 1~8단계 판정. 8단계(경제적 자유)는 조건 충족 시 override.

**수익률 민감도** (거짓 확신 방지): 보수/기본/공격 프리셋(±3%p) + 결과 그래프에 **보수~공격 밴드**(음영)를 항상 함께 표시. 단일 낙관적 곡선 하나만 보여주지 않는다.

**시나리오 A/B 비교**: 최대 5개 저장, 겹쳐보기(점선 오버레이).

**결과 공유 이미지**: canvas로 브랜드 카드(로고·큰 숫자·범위·곡선·목표선·요약칩·컴플라이언스 문구)를 렌더 → 모바일 Web Share 또는 PNG 다운로드. 획득 루프용.

## 5. 데이터 · 인증

- **기본 영속**: localStorage(Zustand persist). 백엔드 없이 완결.
- **Supabase 연결(선택)**: 이메일 OTP 인증 + 로그인 시 원격 동기화(원격 있으면 교체, 없으면 로컬→계정 이관, 이후 디바운스 저장). `.env.local`에 `NEXT_PUBLIC_SUPABASE_URL`/`ANON_KEY` 넣으면 자동 활성화, 없으면 사이드바에 "로컬 모드" 표시.
- **스키마**: `supabase/migrations/0001_init.sql`(profiles/visions/snapshots/engine_buckets/scenarios + RLS + auth 트리거), `0002_tracking.sql`(실천 데이터).
- ⚠️ **미검증 지점**: Supabase 연결 모드는 실제 프로젝트가 있어야 런타임 검증 가능. 코드 경로는 완성했지만 이 세션에서 실제 로그인·동기화까지는 못 돌려봤다. 다음 사람이 붙일 때 여기부터 확인.

## 7. 핵심 결정 (왜 이렇게 했는지)

세션 중 사용자와 함께 내린 결정들 — 다음 사람이 "왜 이랬지?" 헤매지 않도록.

1. **엔진 조립 UX**: react-flow류 자유 캔버스 대신 **자동 배치 흐름그래프**(리스트+슬라이더 하이브리드에서 시작해 최종적으로 SVG 흐름그래프로 발전). 모바일 타겟에 맞춰 조작 단순성 우선. 노드 드래그 이동은 아직 미지원(자동배치라 좌표 저장·연결선 재계산 필요 — 다음 후보).
2. **인스펙터는 오버레이**: 처음엔 3컬럼(팔레트/캔버스/인스펙터) 고정이었으나, "인스펙터 켜면 캔버스가 줄어든다"는 피드백으로 **캔버스 위에 뜨는 오버레이 패널**로 변경. 캔버스 폭은 항상 유지.
3. **거짓 확신 방지**: 가정 수익률 하나로 낙관적 곡선만 보여주면 신뢰 문제 → 보수/기본/공격 프리셋 + 밴드를 결과 패널에 상시 노출.
4. **디자인 시스템 3회 피벗**: Indigo → Plum&Gold → **Charcoal&Yellow-gold**. 매번 "AI가 만든 느낌"·"너무 진하다" 피드백을 반영해 밝기·채도를 조정. 카테고리색(투자=amber/저축=emerald/지출=sky)은 처음부터 고정, 브랜드 액센트가 이와 충돌하지 않도록 매 리컬러마다 확인.
5. **이모지 전면 제거**: 초기엔 이모지 기반 UI였으나, 프로덕트 리뷰에서 미성숙해 보인다는 지적 → 인라인 SVG 아이콘 세트(`Icon.tsx`, 30종)로 전량 교체.
6. **비즈니스 모델은 의도적으로 스코프 아웃**: 사용자가 "고려하지 않아도 된다"고 명시 — 제품 기능·UX 완성도에 집중.
7. **캔버스는 순수 SVG**: eCharts 등 차트 라이브러리 대신 SMIL 애니메이션 기반 자체 구현. CDN 의존 0, 파티클·인터랙션 완전 제어가 이유.

5. **실제 Railway 배포** — 아직 로컬 개발 상태. 배포 파이프라인 미검증.
