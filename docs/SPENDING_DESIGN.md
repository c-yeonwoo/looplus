# E. 지출관리 화면 설계 — v1.2 (Cyrano · PC 웹 우선)

> v1.1 → v1.2: ①**예산 히어로를 변동지출 탭 상단**으로 이동 ②요약 = **변동+고정=총합** + **카테고리 도넛 2개**(변동/고정) ③엔진 연동 설계는 `SPENDING_ENGINE.md`.
> 확정: 또래=연령대만 · 고정 결제일=본인 세팅 · 예산=변동 종합 · 엔진 sync=수동(Phase A).

## 0. 구조 정리 (중요)
- **앱의 '홈'은 대시보드(17·18) 하나뿐.** 지출관리 안엔 홈 없음.
- 지출관리 = 좌측 네비의 한 페이지. 하위 **서브탭: 요약 · 고정지출 · 변동지출**.
- 레이아웃 = 앱 셸(좌측 사이드바) + 메인. PC 폭 기준.

## 1. 목적
최소 마찰 기록 → 시각화 → **또래 + 자기 패턴 진단** → 변동지출(통제 가능한 레버) 조절 유도.

## 2. 핵심 원칙 — 고정 vs 변동 분리 + '통제' 프레이밍
- 고정 = 이미 정해진 것(선 세팅·매달 자동) / 변동 = 내가 조절하는 레버(즉시 기록). 예산은 **변동에만** 적용(종합 한도).

| | 고정지출 | 변동지출 |
|---|---|---|
| 입력 시점 | 선(先) 세팅 → 매달 자동 | 후(後) 기록 → 즉시 |
| PC UX | 테이블 관리(인라인 편집) | **상단 인라인 '빠른 기록 바'** + 즐겨찾기 |

## 3. ① 요약 탭 (총지출 중심)
1. **총합 히어로**: 변동 + 고정 = 이번 달 지출 총합 · 소득 대비 % · 변동/고정 비중 바
2. 고정비 / 변동비 카드 (변동 예산·페이스는 변동 탭으로 안내)
3. **카테고리 도넛 2개**: 변동지출 | 고정지출 (각각 합계·범례)
4. **진단 2종**(§6) — 변동 카테고리·패턴 기준

## 4. ② 변동지출 탭 (예산 + 즉시 기록)
- **상단 예산 히어로:** `n월 현재까지 xxx / 예산` + 페이스 투영 + 예산 수정 (캘린더 월과 동기)
- **인라인 빠른 기록 바:** [금액][카테고리 ▾][날짜(기본 오늘)][메모][저장 = Enter]
- **즐겨찾기 원터치** → 한 번에 기록
- **내역: 캘린더 뷰(디폴트) | 테이블 뷰 토글**
  - **캘린더 뷰:** 월 그리드 · 당일 합계 + 히트맵 띠 · 셀 클릭 시 하단 디테일
  - **테이블 뷰:** 일자 그룹 리스트
- **입력 시간 정책:** 수기 = 시간 없음. 오픈뱅킹(2차) 시 `time` nullable
- (모바일 FAB + 바텀시트 — 이후)

## 5. ③ 고정지출 탭 (세팅형)
- **테이블 관리:** 이름·카테고리·금액·**결제일(본인 세팅)**, 인라인 수정/삭제, "+ 추가" 행
- **오늘 기준 상태:** 결제일 ≤ 오늘 → **결제됨** / 이후 → **예정** (행 배지 · 상단 합계 분리)
- 카테고리 프리셋(주거·통신·구독·보험·대출이자·용돈…)
- 한 번 등록 → 매달 자동. 월 총합 + 소득 대비 % 상단
- (결제일 리마인더 v2)

### 요약과의 관계
- 요약 총합 = 변동(당월) + **결제된 고정만**
- 남은 고정(예정)은 요약 하단 문구로만 표기 (총합 미포함)

## 6. 진단 2종
- **또래 비교(연령대만):** "20대 평균보다 식비 32%↑" — 통계청/카드사 공개 통계 추정('가정' 라벨), 집단 평균
- **내 패턴 인사이트:** "배달 12회(지난달 +40%)", "주말 지출 평일 2배" — 자기 데이터, 즉시 행동가능. 톤=인지→선택

## 7. 정책 / 카피
- 변동 기록 속도 최우선(기본값·즐겨찾기·Enter 저장)
- 또래=집단 평균·가정 라벨
- '통제' 프레이밍: 고정은 정해진 것, 변동은 조절하는 레버
- 엔진 연동: 실측→계획 **수동 반영** (`SPENDING_ENGINE.md`). 자동 덮어쓰기 없음.
