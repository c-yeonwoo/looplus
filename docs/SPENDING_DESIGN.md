# E. 지출관리 화면 설계 — v1.1 (Cyrano · PC 웹 우선)

> v1 대비 변경: ①'홈' 탭 제거(앱 전역 대시보드와 혼동) → 서브탭 **요약·변동지출·고정지출** ②**PC 웹 우선** 레이아웃(FAB·바텀시트 → 인라인 빠른 기록 바). 모바일 반응형은 이후.
> 확정: 또래=연령대만 · 고정 결제일=본인 세팅 · 예산=종합 · 엔진 연동=이후.

## 0. 구조 정리 (중요)
- **앱의 '홈'은 대시보드(17·18) 하나뿐.** 지출관리 안엔 홈 없음.
- 지출관리 = 좌측 네비의 한 페이지. 하위 **서브탭: 요약 · 변동지출 · 고정지출**.
- 레이아웃 = 앱 셸(좌측 사이드바) + 메인. PC 폭 기준.

## 1. 목적
최소 마찰 기록 → 시각화 → **또래 + 자기 패턴 진단** → 변동지출(통제 가능한 레버) 조절 유도.

## 2. 핵심 원칙 — 고정 vs 변동 분리 + '통제' 프레이밍
- 고정 = 이미 정해진 것(선 세팅·매달 자동) / 변동 = 내가 조절하는 레버(즉시 기록). 예산은 **변동에만** 적용(종합 한도).

| | 고정지출 | 변동지출 |
|---|---|---|
| 입력 시점 | 선(先) 세팅 → 매달 자동 | 후(後) 기록 → 즉시 |
| PC UX | 테이블 관리(인라인 편집) | **상단 인라인 '빠른 기록 바'** + 즐겨찾기 |

## 3. ① 요약 탭 (예산 중심)
1. **예산 히어로**: 이번 달 변동지출 vs 종합 예산 + 남은 예산 + **페이스 투영**("이 페이스면 월말 예산 X% 초과 예상")
2. 좌: **카테고리 차트**(도넛+범례) / 우: **진단 2종**(§6)
3. 고정 vs 변동 요약(고정=정해짐/변동=조절)

## 4. ② 변동지출 탭 (PC · 즉시 기록)
- **상단 인라인 빠른 기록 바(항상 노출):** [금액][카테고리 ▾][날짜(기본 오늘)][메모][저장 = Enter] — 키보드로 완결
- **즐겨찾기 원터치:** 프리셋(☕카페 5,000 등) → 한 번에 기록
- **내역: 캘린더 뷰(디폴트) | 테이블 뷰 토글**
  - **캘린더 뷰:** 월 그리드. 각 날짜 셀 = *그날의 합계만* 노출 + 셀 하단 **띠(색 진함 = 상대 지출량 히트맵)**. 많이 쓴 날일수록 진함 → 지출 리듬이 한눈에(주말↑ 등)
  - **셀 클릭 → 캘린더 하단에 '그날 항목' 테이블 펼침**(기본 숨김, 선택 시만). 거기서 항목 수정/삭제/추가
  - **테이블 뷰:** 리스트(날짜·카테고리·메모·금액·수정/삭제, 일자 그룹)
  - 흐름: **캘린더 → 빠른 기록(날짜 추가) → 날짜 셀 디테일 테이블(선택 시만)**
- **입력 시간 정책:** 수기 입력은 **시간 세팅 없음**(기록 시각 자동 or 생략). 오픈뱅킹/마이데이터 연동(2차) 시 **실제 지출 시각 자동** → `SpendingLog.time = nullable`
- **오픈뱅킹/마이데이터:** PC 웹에서도 가능(계좌 인증 기반). 지출 일괄 연동 = 2차
- (모바일 반응형 시 FAB + 바텀시트 — 이후)

## 5. ③ 고정지출 탭 (세팅형)
- **테이블 관리:** 이름·카테고리·금액·**결제일(본인 세팅)**, 인라인 수정/삭제, "+ 추가" 행
- 카테고리 프리셋(주거·통신·구독·보험·대출이자·용돈…)
- 한 번 등록 → 매달 자동. 총합 + 소득 대비 % 상단
- (결제일 리마인더 v2)

## 6. 진단 2종
- **또래 비교(연령대만):** "20대 평균보다 식비 32%↑" — 통계청/카드사 공개 통계 추정('가정' 라벨), 집단 평균
- **내 패턴 인사이트:** "배달 12회(지난달 +40%)", "주말 지출 평일 2배" — 자기 데이터, 즉시 행동가능. 톤=인지→선택

## 7. 정책 / 카피
- 변동 기록 속도 최우선(기본값·즐겨찾기·Enter 저장)
- 또래=집단 평균·가정 라벨
- '통제' 프레이밍: 고정은 정해진 것, 변동은 조절하는 레버
- 엔진/B 자동 연동은 이후(수기 유지)
